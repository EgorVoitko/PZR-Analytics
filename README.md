# PZR Analytics

Аналітичний дашборд та система управління продажами для онлайн-магазину гри ARK: Survival Ascended.

---

## Про проєкт

PZR Analytics — це внутрішній інструмент для управління онлайн-ігровим магазином. Надає адміністраторам та працівникам доступ до статистики продажів у реальному часі, аналітики клієнтів, графіків доходів та публічної сторінки оплати з інтеграцією Stripe.

Система підтримує дві ролі — **Адміністратор** та **Працівник** — кожна з власним інтерфейсом і набором прав доступу.

---

## Функціональність

### Адміністратор
- Головний дашборд із графіками доходів (тиждень / місяць / рік), розподілом заробітків за категоріями та порівнянням місяців
- Повна історія продажів із фільтрацією за працівником і категорією, сортуванням та пагінацією
- Управління працівниками — перегляд статистики, створення нових акаунтів напряму з дашборду
- База клієнтів із системою рівнів (Gold, Silver, Bronze, New) на основі суми витрат
- Додавання, редагування та видалення записів про продажі
- Експорт будь-якої сторінки у файл Excel (.xlsx)

### Працівник
- Особистий дашборд з індивідуальною статистикою та графіками продажів
- Доступ до власної історії продажів
- Особисте посилання на оплату — унікальний URL, який прив'язує будь-яку покупку до цього працівника

### Сторінка оплати (`/pay`)
- Публічна сторінка, вхід не потрібен
- Товари розподілені за категоріями (Dino, Boss, Weapon, Armor тощо)
- Інтегровано з **Stripe Checkout** (тестовий режим)
- Після успішної оплати запис про продаж автоматично створюється в базі даних
- Якщо сторінку відкрито через особисте посилання працівника — продаж зараховується йому

### Оновлення в реальному часі
- Таблиця продажів оновлюється миттєво при додаванні нового запису
- Статистика клієнтів оновлюється в реальному часі після оплати

## Схема бази даних

Проєкт використовує 4 основні таблиці:

- **profiles** — акаунти користувачів із ролями (`admin` / `employee`), автоматично створюються при реєстрації через тригер
- **products** — каталог магазину з категоріями, цінами та кількістю товару
- **customers** — записи клієнтів із кількістю покупок, загальними витратами, знижкою та рівнем
- **sales** — записи транзакцій, прив'язані до працівника, товару та клієнта

Row Level Security (RLS) увімкнено для всіх таблиць. Працівники бачать лише свої продажі; адміністратори мають доступ до всього.

## Запуск локально

Клонувати репозиторій:

```bash
git clone https://github.com/EgorVoitko/PZR-Analytics.git
cd PZR-Analytics
```

Встановити залежності:

```bash
npm install
```

Створити файл `.env` на основі `.env.example` та заповнити змінні:

```env
SUPABASE_URL=
SUPABASE_KEY=
SUPABASE_SERVICE_KEY=

STRIPE_SECRET_KEY=
STRIPE_PUBLISHABLE_KEY=
```

Запустити development server:

```bash
npm run dev
```

Відкрити локальний застосунок:

```
http://localhost:3000
```

Зібрати production build:

```bash
npm run build
```

## Процес оплати

```
Клієнт відкриває /pay?staff=<uuid>
        ↓
Обирає товар → вводить ім'я/email → натискає Pay
        ↓
POST /api/create-checkout  (серверний маршрут Nuxt)
        ↓
Сторінка Stripe Checkout  (тестова карта: 4242 4242 4242 4242)
        ↓
POST /api/confirm-payment  → створює запис про продаж у Supabase
        ↓
Продаж з'являється в дашборді в реальному часі
```
