# PZR Analytics

Аналітичний дашборд та система управління продажами для онлайн-магазину гри ARK: Survival Ascended.
Демо - https://pzr.netlify.app/
---

## Про проєкт

PZR Analytics — це внутрішній інструмент для управління онлайн-ігровим магазином. Надає адміністраторам та працівникам доступ до статистики продажів у реальному часі, аналітики клієнтів, графіків доходів та публічної сторінки оплати з інтеграцією Stripe.

Система підтримує дві ролі — **Адміністратор** та **Працівник** — кожна з власним інтерфейсом і набором прав доступу.

---

## Функції

- Аналітичний дашборд з графіками доходів за тиждень, місяць та рік із порівнянням з попереднім місяцем.
- CRUD для продажів: створення, редагування та видалення записів з вибором працівника, товару, кількості та дати.
- Управління працівниками: перегляд індивідуальної статистики, створення нових акаунтів напряму з адмін-панелі.
- База клієнтів із системою лояльності — автоматичне присвоєння рівнів Gold, Silver, Bronze, New на основі суми витрат.
- Публічна сторінка оплати з каталогом товарів, розподілених за категоріями, та інтеграцією Stripe Checkout.
- Особисте посилання для кожного працівника — будь-яка покупка за цим посиланням автоматично зараховується йому.
- Оновлення даних у реальному часі через Supabase Realtime — нові продажі та зміни клієнтів відображаються без перезавантаження.
- Фільтрація продажів за працівником і категорією товару, сортування та пагінація таблиці.
- Експорт будь-якої сторінки у файл Excel (.xlsx) одним натисканням.
- Авторизація з захищеними маршрутами та розмежуванням прав: адміністратор бачить усю статистику, працівник — лише свої дані.

## Схема бази даних

Проєкт використовує 4 основні таблиці:

- **profiles** — акаунти користувачів із ролями (`admin` / `employee`), автоматично створюються при реєстрації через тригер
- **products** — каталог магазину з категоріями, цінами та кількістю товару
- **customers** — записи клієнтів із кількістю покупок, загальними витратами, знижкою та рівнем
- **sales** — записи транзакцій, прив'язані до працівника, товару та клієнта

Row Level Security (RLS) увімкнено для всіх таблиць. Працівники бачать лише свої продажі; адміністратори мають доступ до всього.

## Запуск локально

Клонувати репозиторій:

```bash
git clone https://github.com/EgorVoitko/PZR-Analytics.git
cd PZR-Analytics
```

Встановити залежності:

```bash
npm install
```

Створити файл `.env` на основі `.env.example` та заповнити змінні:

```env
SUPABASE_URL=
SUPABASE_KEY=
SUPABASE_SERVICE_KEY=

STRIPE_SECRET_KEY=
STRIPE_PUBLISHABLE_KEY=
```

Запустити development server:

```bash
npm run dev
```

Відкрити локальний застосунок:

```
http://localhost:3000
```

Зібрати production build:

```bash
npm run build
```

## Сторінка оплати

Публічна сторінка доступна за адресою `/pay` без авторизації. Покупець обирає категорію товару, конкретний товар та кількість, вводить ім'я та email, після чого перенаправляється на Stripe Checkout.

Якщо сторінку відкрито за посиланням працівника (`/pay?staff=<uuid>`), продаж автоматично зараховується цьому працівнику.

Тестові дані картки Stripe:

| Поле | Значення |
|---|---|
| Номер картки | `4242 4242 4242 4242` |
| Термін дії | Будь-яка майбутня дата |
| CVC | Будь-які 3 цифри |

Технічний флоу:

```
/pay?staff=<uuid>
        ↓
Вибір товару → ім'я / email → Pay
        ↓
POST /api/create-checkout
        ↓
Stripe Checkout
        ↓
POST /api/confirm-payment → запис у Supabase
        ↓
Продаж з'являється в дашборді в реальному часі
```
