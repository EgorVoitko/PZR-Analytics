# PZR Analytics

PZR Analytics — це внутрішній дашборд для управління онлайн-магазином. Об'єднує аналітику продажів, базу клієнтів, управління командою та публічну сторінку оплати в єдиній системі.

Демо — https://pzr.netlify.app/

## Огляд

PZR Analytics вирішує задачу обліку продажів у команді з кількох менеджерів, де кожен продає товари різним клієнтам і важливо бачити як загальну картину, так і особисту ефективність. Система автоматично атрибутує продажі конкретному менеджеру — через особисте посилання або ручний запис — і відображає аналітику в реальному часі.

Продукт орієнтований на невеликі магазини, де адміністратор стежить за загальними показниками, а кожен менеджер бачить лише свій особистий прогрес.

## Бізнес-функції

- Аналітичний дашборд із метриками виручки, кількості продажів, сьогоднішнього доходу та активних клієнтів.
- Графіки доходу за тиждень, місяць та рік із накладенням попереднього місяця для порівняння динаміки.
- Розбивка доходу за категоріями товарів у вигляді donut-графіка.
- Стрічка останніх продажів у реальному часі без перезавантаження сторінки.
- CRUD для продажів: створення, редагування та видалення з фільтрацією, сортуванням і пагінацією.
- Особистий дашборд менеджера з власною аналітикою, відокремленою від загальної.
- База клієнтів із системою лояльності: автоматичне присвоєння рівнів Gold, Silver, Bronze і New на основі суми витрат.
- Профіль клієнта з реальною історією покупок із бази даних — назви товарів, дати, кількість і суми — та графіками витрат за тиждень, місяць і рік.
- Управління командою: перегляд статистики кожного менеджера, створення нових акаунтів безпосередньо з адмін-панелі.
- Публічна сторінка оплати для покупців без авторизації з каталогом товарів і Stripe Checkout.
- Сторінка підтвердження оплати відображає назву придбаного товару та фінальну суму отриману від Stripe.
- Особисте посилання менеджера — покупка через `/pay?staff=<uuid>` автоматично зараховується йому.
- Експорт будь-якої сторінки у файл Excel одним натисканням.
- Discord-сповіщення при кожному новому продажу через Stripe — назва товару, сума, покупець і менеджер одразу в канал магазину.

## Технічні функції

- Nuxt 3 single page application на Vue 3 і TypeScript з вимкненим SSR.
- Supabase Auth для email/password авторизації та керування сесією.
- Supabase Postgres із Row Level Security — менеджери бачать лише свої продажі, адміністратори мають доступ до всього.
- Supabase Realtime для миттєвого відображення нових продажів і змін клієнтів без polling.
- Резервний polling кожні 15 секунд на випадок розриву Realtime-з'єднання.
- Nitro server routes для Stripe Checkout і реєстрації акаунтів менеджерів.
- Серверна валідація вхідних даних у всіх API ендпоінтах — типи, обов'язкові поля та діапазони значень.
- Серверне логування помилок із контекстом для діагностики збоїв Stripe та Supabase.
- Stripe Checkout для безпечної обробки платежів.
- Pinia store для стану авторизації та module-level сingletons для persistентного кешу даних між навігацією.
- Chart.js із vue-chartjs для інтерактивних графіків.
- SheetJS для генерації Excel-файлів на клієнті.
- Глобальний middleware для захисту маршрутів і редиректів залежно від ролі.
- Discord webhook — після підтвердження оплати Nitro надсилає embed у канал магазину: товар, кількість, сума, покупець і менеджер.

## Сторінки

| Маршрут | Сторінка | Призначення |
| --- | --- | --- |
| `/login` | Login | Вхід через email і пароль. |
| `/` | Admin Dashboard | Загальна аналітика: виручка, продажі, клієнти, графіки та стрічка подій. |

| `/dashboard/sales` | Sales | Таблиця всіх продажів із CRUD, фільтрами, сортуванням і пагінацією. |
| `/dashboard/employees` | Employees | Список менеджерів із індивідуальною статистикою. |
| `/dashboard/employees/[id]` | Employee Profile | Профіль менеджера з деталями та власними продажами. |
| `/dashboard/customers` | Customers | База клієнтів із рівнями лояльності та пошуком. |
| `/dashboard/customers/[id]` | Customer Profile | Профіль клієнта: реальна історія покупок із БД, графіки витрат за тиждень/місяць/рік, рівень лояльності та знижка. |
| `/pay` | Payment | Публічна сторінка оплати без авторизації з Stripe Checkout. |

## Модель даних

PZR Analytics використовує 4 таблиці Supabase:

- `profiles`: акаунти користувачів із роллю `admin` або `employee`, автоматично створюються тригером при реєстрації.
- `products`: каталог товарів із категоріями, SKU, цінами та кількістю. Містить понад 300 позицій ARK: Survival Ascended.
- `customers`: записи покупців із кількістю покупок, загальними витратами, знижкою та рівнем лояльності.
- `sales`: транзакції, прив'язані до менеджера (`staff_id`), товару (`product_id`) і клієнта (`customer_id`). Зберігає кількість, суму та статус оплати.

Рівень лояльності клієнта обчислюється динамічно: Gold — від $1 000, Silver — від $500, Bronze — від $250.

## Флоу оплати

```text
/pay?staff=<uuid>
        ↓
Вибір товару → ім'я / email → Pay
        ↓
POST /api/create-checkout  (валідація + Stripe session)
        ↓
Stripe Checkout
        ↓
POST /api/confirm-payment  (перевірка статусу + запис у Supabase)
        ↓
Discord webhook → сповіщення в канал магазину
        ↓
Продаж з'являється в дашборді в реальному часі
```

## Напрям розвитку

Поточна версія закриває повний цикл продажу: від оплати покупцем до аналітики в реальному часі та сповіщення команди. Подальший розвиток може включати:

**Сповіщення та комунікація**
- Discord-бот для прямих повідомлень покупцям після оплати — поле `discord_id` у таблиці `customers` вже зарезервовано під цю інтеграцію.
- Автоматичні щотижневі звіти для адміністратора в Discord або на email — підсумок виручки, топ менеджерів і топ товарів за тиждень.

**Аналітика та геймофікація**
- Лідерборд менеджерів — рейтинг за виручкою тижня або місяця з місцями та медалями, доступний усій команді.
- Розширена аналітика: прогноз виручки на наступний місяць на основі накопиченої історії, рейтинг товарів за популярністю та маржинальністю.

**Маркетинг і клієнти**
- Система промокодів — адміністратор створює коди зі знижкою, покупець вводить на сторінці оплати, знижка застосовується автоматично.
- QR-код особистого посилання — менеджер генерує QR на свій `/pay?staff=<uuid>` одним кліком і ділиться ним у Discord або соцмережах.
- PDF-квитанція після оплати — автоматично надсилається на email покупця з деталями замовлення.

**Інфраструктура**
- Підтримка кількох магазинів в одному акаунті з окремими каталогами, командами та аналітикою.
- Мобільна адаптація інтерфейсу для роботи з дашборду зі смартфона.

## Запуск локально

Встановити залежності:

```bash
npm install
```

Створити файл `.env` та заповнити змінні:

```env
SUPABASE_URL=
SUPABASE_KEY=
SUPABASE_SERVICE_KEY=

STRIPE_SECRET_KEY=
STRIPE_PUBLISHABLE_KEY=

# Опційно — Discord webhook для сповіщень про нові продажі
DISCORD_WEBHOOK_URL=
```

Запустити development server:

```bash
npm run dev
```

Відкрити локальний застосунок:

```text
http://localhost:3000
```

Зібрати production build:

```bash
npm run build
```
